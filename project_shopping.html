쇼핑몰

쇼핑몰의 회원 관련, 제품 관련, 장바구니, 주문, 리뷰 기능 구현이 목표입니다.
Spring Framework를 이용할 때 다소 복잡했던 초기 설정을 간편하게 처리해주는 Spring Boot의 장점을 느낄 수 있었습니다. 또한 JPA를 사용하기 위해 학습하는 데 시간이 다소 걸리긴 했으나, Mybatis를 사용할 때 직접 작성하던 쿼리를 메서드만으로 구현해볼 수 있었습니다.

블로그 링크 깃허브 링크

구현기능

  1. 회원
    • 회원가입
    • 로그인
    • 배송지
  2. 제품
    • 제품 등록
    • 제품 리스트
    • 제품 조회
    • 제품 품절
  3. 장바구니
    • 장바구니 CRUD
  4. 주문
    • 주문
    • 주문 조회
  5. 리뷰(추가예정)
    • 리뷰 작성/수정/삭제
    • 리뷰 조회

1. 회원

  • 회원가입
    • 유효성 검사
    • BCryptPasswordEncoder 비밀번호 암호화
    • 이메일 인증
  • 로그인
    • Spring Security를 이용한 로그인
  • 회원페이지
    • 접근 권한 체크
  • 배송지 조회
    • 배송지(우편번호, 주소, 상세주소) 조회
    • db에 배송지 존재하지 않을 경우 [등록]
    • 존재할 경우 [변경]
  • 배송지 등록/변경
    • 우편번호 찾기 (Daum 주소 API 이용)

2. 제품

  • 제품 등록
    • 하위 카테고리 리스트 불러옴
    • 제품명, 제품 가격, 제품 판매상태, 썸네일이미지, 제품 소개 작성
    • 제품 소개는 웹 에디터인 SummerNote 이용
  • 제품 리스트
    • 상위 카테고리(하위 카테고리에 속한 제품 모두 조회)
    • 하위 카테고리
  • 제품 조회
    • 제품 정보(썸네일, 제품명, 가격, 소개) 조회
    • 수량 입력 및 장바구니 담기 / 바로 구매 버튼
  • 제품 품절
    • 제품 리스트 페이지 품절 표시
    • 제품 조회 페이지 품절 표시 및 장바구니, 구매 버튼 X
    • 이미 장바구니에 존재할 경우 품절 표시 및 제외
    • 주문 페이지에 품절 상품 제외

3. 장바구니

  • 장바구니
    • 회원 [DB] / 비회원 [Session]에 저장
    • 제품별 구매 수량 변경/삭제
    • 수량에 따라 금액 표시

4. 주문

  • 주문 공통
    • 주문정보 입력(주문자명, 전화번호, 배송지)
    • 회원의 경우, 등록된 배송지를 보여주며 직접 입력할 수 있도록 함
    • 주문 성공 시, 주문 완료 페이지로 이동
  • 장바구니 → 주문
    • 장바구니에 담긴 제품 리스트
    • 주문 완료 시, 장바구니에 담긴 제품 삭제
  • 바로구매 → 주문
    • 해당 제품만 구매
    • 장바구니 변화 없음
  • 주문 내역 조회
    • 비회원 : 주문자명, 주문번호 입력하여 조회
    • 회원 : 주문한 내역 리스트 → 상세보기
